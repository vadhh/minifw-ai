## 3. Demo Verification — Hospital Sector

USB package `dist/minifw-usb-hospital-v2.2.0/` verified against the same standard.

**Finding: 2 blocking issues.**

### 3.1 Missing `healthcare_threats.txt` feed (CRITICAL)

The hospital engine startup log showed:
```
[FEEDS] Sector feed not found: /opt/minifw_ai/config/feeds/healthcare_threats.txt
[SECTOR_LOCK] Loaded 0 sector-specific feed patterns
```

The hospital attack domains used by the injector (`lockbit-blog.com`, `medrecords-transfer.io`, `RyukReadMe.example.com`, etc.) were not being matched by any feed. Result: only `burst_behavior` events at score=10 — no BLOCK events, no `dns_denied_domain` reason.

**Fix:** Copied `config/feeds/healthcare_threats.txt` from the main repo into the USB bind-mount directory (`dist/minifw-usb-hospital-v2.2.0/config/feeds/`). Engine restarted and confirmed:
```
[FEEDS] Loaded 11 patterns from healthcare_threats.txt
[SECTOR_LOCK] Loaded 11 sector-specific feed patterns
```

### 3.2 BLOCK events not firing (caused by 3.1)

Once feeds loaded, BLOCK events appeared immediately:
```
[block] domain=[REDACTED] score=50 reasons=['dns_denied_domain', 'burst_behavior'] seg=mednet
```

Scoring: `dns_denied(+40) + burst(+10) = 50 > mednet block_threshold(45)` → BLOCK confirmed.

**Post-fix event summary (hospital):**

|---|---|---|
| allow | ✓ | — |
