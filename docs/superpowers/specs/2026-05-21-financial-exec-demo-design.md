## Goal

A one-button, reproducible executive presentation of MiniFW-AI in the finance sector. Requirements: no terminal noise, clean startup, automatic browser launch, trusted TLS, predictable BLOCK event within 2 minutes, graceful shutdown.

## TLS Setup (`setup_tls.sh`)

Runs **once before the first demo**. No internet required after initial tool setup.

1. **Generate local CA** — `openssl req` produces `certs/minifw-ca.key` + `certs/minifw-ca.crt` (CN: "MiniFW Demo CA", 825-day validity, `basicConstraints=CA:true`).
2. **Issue server cert** — signed by that CA, valid for `localhost` + SAN `127.0.0.1`.
3. **Install to system trust store** — `sudo cp minifw-ca.crt /usr/local/share/ca-certificates/ && sudo update-ca-certificates`. Covers curl, Python requests, Chromium-based browsers.
4. **Install to NSS** — iterates `~/.pki/nssdb` and any Firefox profile `cert9.db` via `certutil -A`. Covers Firefox and Chrome NSS users.

`teardown_demo.sh` reverses steps 3 and 4 cleanly.

`run_demo.sh` aborts with a single clear error line if `certs/server.crt` is missing.

## Demo Scheduler (`scheduler/demo_scheduler.py`)

A standalone Python process. Writes synthetic DNS events in dnsmasq log format (`dnsmasq[1]: query[A] domain.com from 10.50.0.1`) to `logs/demo.dnsmasq.log`. The engine is configured with `MINIFW_DNS_SOURCE=file` pointing to that file and tail-follows it via `stream_dns_events_file()`. Events flow through the full scoring pipeline — real BLOCK decisions, no bypass.

### Timeline

| Time | Phase | Traffic |
|------|-------|---------|
| T+0s → T+60s | Normal | Bloomberg terminal, Reuters feed, SWIFT gateway, `trading.corp` DNS — score <45 (green) |
| T+60s → T+80s | Attack sequence | Four events in rapid succession (see below) |
| T+90s+ | Post-block normal | Normal traffic resumes — firewall running cleanly |

### Attack Sequence (T+60s–T+80s)

All events target `10.50.0.1` in the `trading` segment (block threshold: 80).

| Event | Domain | Score contributors |
|-------|--------|--------------------|
| Tor exit node query | `tor-exit-4f2a.net` | DNS feed +40, ASN deny +15 |
| Crypto C2 beacon | `xmr-pool-hidden.onion.to` | DNS feed +40, YARA +35 |
| Card data exfil probe | `cc-dump-api.ru` | DNS feed +40, MLP +30 |
| PCI boundary violation | `pastebin.com` from trading subnet | SNI +35, burst +10 |

Cumulative score exceeds 80 → engine emits `"final_action": "block"` → dashboard BLOCK alert fires.

## `run_demo.sh` — Startup Sequence

1. **Pre-flight** — verify `certs/server.crt` exists; verify Python 3 + uvicorn available.
2. **Environment** — export: `DEMO_MODE=1`, `PRODUCT_MODE=minifw_financial`, `MINIFW_SECTOR=finance`, `AI_ENABLED=1`, `MINIFW_DISABLE_FLOWS=1`, `MINIFW_DNS_SOURCE=file`, `MINIFW_DNS_LOG=logs/demo.dnsmasq.log`, `MINIFW_SECRET_KEY`, `MINIFW_ADMIN_PASSWORD=Finance1!`.
3. **Terminal output suppressed** — engine and web log to `logs/engine.log` and `logs/web.log`. Terminal shows exactly:
   ```
   [minifw] Starting Financial Demo...
   [minifw] Engine started (PID XXXX)
   [minifw] Dashboard ready → https://localhost:8443  (admin / Finance1!)
   [minifw] Press Ctrl+C to stop.
   ```
4. **Engine** — `python3 app/minifw_ai/main.py > logs/engine.log 2>&1 &`
5. **Web** — `uvicorn ... --port 8443 --ssl-keyfile certs/server.key --ssl-certfile certs/server.crt --log-level warning > logs/web.log 2>&1 &`
6. **Health poll** — `curl -s --cacert certs/minifw-ca.crt https://localhost:8443/health` up to 20s; abort with log hint on timeout.
7. **Scheduler** — `python3 scheduler/demo_scheduler.py > logs/scheduler.log 2>&1 &`
8. **Browser** — `xdg-open https://localhost:8443` (Linux) or `open https://localhost:8443` (macOS), suppressed, best-effort.

## Graceful Shutdown

`trap cleanup EXIT INT TERM` kills engine PID, web PID, scheduler PID in order. Prints `[minifw] Demo stopped.` — one line. Logs retained for post-demo review.

## Scope Boundaries

- No Docker dependency — Python venv only.
- No root required at runtime — `DEMO_MODE=1` skips nftables/ipset.
- Root required once: `setup_tls.sh` for CA trust-store install.
- No manual presenter action needed for BLOCK — fully automatic.
- `teardown_demo.sh` removes CA from trust stores post-meeting.
