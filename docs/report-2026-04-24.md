## 2. Feature — Financial Sector Detection Rules

**Files created:**
- `app/minifw_ai/sector_rules/__init__.py` — dispatcher entry point
- `app/minifw_ai/sector_rules/financial_rules.py` — 10 detection functions
- `app/minifw_ai/sector_rules/financial_policy.py` — policy decision engine

### Detection Functions

| Function | Description |
|----------|-------------|
| `detect_high_risk_country` | Blocks traffic destined for OFAC-sanctioned jurisdictions (KP, IR, SY, RU) |
| `detect_blocked_asn` | Enforces ASN-level deny feed matches |
| `detect_unknown_external_asn` | Flags outbound to ASNs not in the known-good allow list |
| `detect_suspicious_port` | Alerts on non-standard ports associated with C2 and data exfiltration |
| `detect_large_outbound_transfer` | Triggers at 50 MB (alert) and 200 MB (critical block) |
| `detect_sensitive_asset_exfil` | Monitors internal asset IP ranges for abnormal outbound volume |
| `detect_after_hours_external_activity` | Flags outbound from internal assets outside business hours (07:00–19:00) |
| `detect_tls_anomaly` | Detects missing TLS on external connections or absent SNI |
| `detect_api_anomaly` | Scores against known-good API endpoint patterns (auth failures, schema violations) |
| `detect_connection_burst` | Triggers at ≥80 connections/minute — beaconing/scanning heuristic |

### Policy Decision Logic

`decide_financial_action()` resolves detections to a final action (`block / alert / monitor`):

- **Immediate block:** Any detection of type `blocked_asn`, `very_large_outbound_transfer`, `sensitive_asset_exfiltration_risk`, `api_schema_violation`, `api_auth_failure_burst`, or `high_risk_country`
- **Block:** Two or more HIGH/CRITICAL severity detections (combined risk)
- **Alert:** Aggregate confidence ≥ 0.75
- **Monitor:** All other cases with detections

### Design Notes

- All rule functions use `.get()` for flow field access — handles today's conntrack-level data without requiring enriched fields (dst_country, JA3, full API metadata)
- `FINANCIAL_DEFAULTS` is deep-copied per evaluation to prevent module-level set mutation across concurrent calls
- `detect_suspicious_port` guards against `None` dst_port before membership test

## 4. Feature — Engine Integration Hook

**File:** `app/minifw_ai/main.py`

A post-scoring hook evaluates sector rules after the base scoring pipeline completes. The sector rules layer can only **upgrade** an action (allow→monitor, allow/monitor→block); it cannot downgrade a base pipeline decision.

```
DNS event → [base pipeline: feeds / burst / MLP / YARA / score_and_decide]
          → [sector hook: evaluate_sector() → decide_sector_action()]
          → final action: block / monitor / allow
```

The hook is guarded by a try/except import block — if `sector_rules` is missing, the engine runs without sector enrichment and logs a warning at startup only.

A `_build_sector_flow()` helper bridges the base pipeline's internal state (conntrack flows, burst counters) into the flat dict format the sector rules expect.

## 9. Fix — `pump_zeek` StopIteration Spam

**File:** `app/minifw_ai/main.py`

**Problem:** `next(zeek_iter)` raised `StopIteration` when Zeek ssl.log was absent (iterator immediately exhausted). The generic `except Exception` handler caught it, logged a WARNING, and re-entered the loop — producing thousands of log lines per second and burning CPU.

**Fix:** Added `except StopIteration: break` before the generic except block. Iterator exhaustion exits the pump cleanly with no log output.

---

## 10. Fix — Zeek Graceful Degradation

**File:** `app/minifw_ai/collector_zeek.py`

**Problem:** `stream_zeek_sni_events()` called `path.open()` even when ssl.log didn't exist, raising `FileNotFoundError` on every `next()` call in the pump loop.

**Fix:** Check `Path(log_path).exists()` at entry. If absent, log once at INFO level and `return` (empty iterator). SNI enrichment is disabled silently; the engine continues normally.

This fix applies to all sectors — not just financial.
