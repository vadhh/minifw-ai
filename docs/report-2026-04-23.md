## 2. Milestone — Standalone Hospital Demo (v3)

**Package:** `dist/minifw-ai-usb-v2.2.0v3/`

**Key Improvements over v2 (Docker):**
- **Zero Friction:** No Docker or Docker-Compose dependency. Runs on any machine with Python 3.
- **Instant Start:** Eliminated the 3-minute `docker load` latency. Demo starts in < 2 seconds.
- **Offline Reliability:** Fully self-contained folder including core engine, web app, and AI models.
- **Transparency:** Simulation data is visible in `demo_data/` for easy explanation to stakeholders.

## 3. Feature — `DEMO_MODE` Simulation

**File:** `app/minifw_ai/main.py`

Implemented a high-fidelity traffic simulator that cycles through `normal_traffic.json` and `attack_traffic.json`. 

**Simulated Scenarios:**
1. **Normal Operations:** Standard healthcare domain traffic (EMR, PACS systems).
2. **Ransomware C2:** Detection of malicious domains tied to ransomware campaigns.
3. **Suspicious API Leak:** Identifying unauthorized data exfiltration via API endpoints.
4. **Anonymizer Usage:** Blocking access via known Tor exit nodes.

## 4. Fix — Automated Database Provisioning

**File:** `app/database.py`

**Problem:** Fresh demo deployments failed authentication because the database was empty.
**Solution:** Added logic to `init_db()` to automatically create an `admin` user on first run.

**Hardenings applied:**
- Provisioned with `admin@hospital.local` and hashed `Hospital1!`.
- Explicitly set `sector="hospital"` and `must_change_password=False` for demo stability.
- Fixed `IntegrityError` by ensuring all mandatory HIPAA-compliant fields (email, sector) are populated.

---

## 5. Fix — DataTables Column Count Error

**File:** `app/web/templates/admin/events.html`

**Problem:** "DataTables warning: table id=eventsTable - Incorrect column count". This occurred when the event log was empty or during specific responsive triggers.
**Fix:** 
1. Removed manual `{% else %}` colspan rows that conflicted with DataTables' own empty-state management.
2. Explicitly defined column widths and attributes in the JS initialization for a perfect 1:1 match with the HTML `<thead>`.

---

## 6. Fix — Audit Log Localization

**File:** `run_demo.sh`

**Problem:** Permission denied errors when trying to write to `/opt/minifw_ai/logs` (system default).
**Solution:** Set `MINIFW_AUDIT_LOG=logs/audit.jsonl` in the launcher script. All logs are now captured locally within the portable demo folder.
