## 2. Gap Closure Implementation

### 2.1 Event Dataclass â€” New Typed Fields

**File:** `app/minifw_ai/events.py`

Added three optional boolean fields to the `Event` dataclass (after `decision_owner`):

| Field | Purpose |
|---|---|
| `student_flagged` | Set when src IP falls within a student subnet |
| `vpn_block_enforced` | Set when a VPN/proxy YARA rule fired on the event |
| `audit_mode` | Set when `strict_logging` is active (event written regardless of action) |

Default `False` on all fields â€” zero impact on non-education sectors.

### 2.2 `_ip_in_subnets` Helper

**File:** `app/minifw_ai/main.py`

Added module-level pure helper before `def run()`:

```python
def _ip_in_subnets(ip: str, subnets: list[str]) -> bool:
    try:
        addr = ipaddress.ip_address(ip)
        return any(addr in ipaddress.ip_network(s, strict=False) for s in subnets)
    except (ValueError, TypeError):
        return False
```

`TypeError` included to handle IPv6 addresses checked against IPv4 subnets.

### 2.3 Startup Hook â€” `force_safesearch`, `block_vpns`, `block_proxies`

**File:** `app/minifw_ai/main.py` â€” startup block (after Tor loading)

- `force_safesearch=True`: extends `feeds.allow_domains` with both bare domain and wildcard (`google.com` + `*.google.com`) for all 4 safesearch domains â€” 8 patterns total. Prevents these domains from ever being blocked by a feed entry.
- `block_vpns` / `block_proxies`: logs confirmation that VPN/proxy entries are active via `school_blacklist.txt` feed.

### 2.4 Event Write Path â€” `log_student_activity`, `block_vpns`, `strict_logging`

**File:** `app/minifw_ai/main.py` â€” event write path (after `redact_payloads`)

- `log_student_activity=True`: checks src IP against `segment_subnets.student` via `_ip_in_subnets`; sets `ev.student_flagged = True` on match.
- `block_vpns` / `block_proxies`: checks `ev.reasons` for any entry containing `"VpnProxy"`; sets `ev.vpn_block_enforced = True`.
- `strict_logging=True`: sets `ev.audit_mode = True` and writes the event regardless of action (normally only `monitor` and `block` are written).
