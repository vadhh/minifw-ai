## 3. Priority 1 — Fresh Machine Validation

### Protocol Document
**File:** `docs/FRESH_MACHINE_VALIDATION.md`

Step-by-step cold-start protocol for a fresh VM or physical machine.
Covers: VM spec (Ubuntu 22.04 LTS, 2 vCPU, 4 GB), per-sector test checklists
(hospital standalone + all four Docker packages), YARA validation command,
results table, failure triage guide, and a hard distribution gate rule:

> **Do not distribute a package that has not completed this protocol on a fresh VM.**

### Automated Pre-Flight Checker
**File:** `dist/minifw-usb-hospital-standalone-v2.2.0/validate_install.sh`

Runs in ~5 seconds. Checks 26 items across 7 categories:
- Python version (≥ 3.10)
- Virtual environment (activate + import fastapi / uvicorn / sqlalchemy / bcrypt / yara)
- Port 8000 free
- 12 core files present (including requirements.txt)
- YARA rules: count, compile check
- Logs directory writable, clean state
- policy.json valid JSON + sector field correct
- App import smoke test

Exits 0 on pass (with warnings), 1 on any failure.

### Container Cold-Start Test — Run 1 (failed — bugs found)

**Environment:** Ubuntu 22.04 Docker container, no Python pre-installed, volume-mounted hospital standalone package.

**Run 1 result: 3 failures**

| Failure | Root Cause |
|---------|-----------|
| `requirements.txt: No such file or directory` | File was never included in the dist package |

2. Deleted `sme_rules.yar` and `test_rules.yar` from hospital `yara_rules/`
3. Fixed `validate_install.sh` policy.json sector detection (keys are `_mode`/`_sector`, not `product_mode`/`sector`)

## 6. Priority 4 — Client Q&A

**File:** `docs/CLIENT_QA.md`

Technical answers to the six standard questions asked by buyers:

| Question | Answer summary |
|----------|---------------|
| "Does this inspect encrypted traffic?" | DNS-first detection before TLS handshake; SNI extracted in plaintext; flow + ASN for direct-IP fallback |
| "What about false positives?" | Additive scoring requires multiple signals to reach block threshold; one-click release; 24h auto-expire; per-segment whitelist |
| "Can this run offline?" | All feeds, model, YARA rules cached locally at startup; no cloud dependency; feed updates via USB |
| "Can this work without agents?" | Gateway-only; DNS observation covers every device; critical for IoMT (ventilators, infusion pumps) |
| "How fast is blocking?" | 200ms–800ms from DNS query to nftables DROP; hard gates bypass AI steps for volumetric attacks |
| "Can logs be exported?" | JSONL event stream + audit trail; Filebeat/Fluentd compatible; trace IDs as SIEM correlation key |

Each answer includes: short answer (1 sentence), full technical explanation, and "what this means in practice" framing for the sales conversation.
